Flow Animation

The first row has slack. Work can arrive unevenly and still move through the constraint. The second row runs close to saturation. The same unevenness becomes a queue.

Ready Constraint with slack Done
Ready Constraint saturated Done

This is deliberately schematic. The point is not the exact animation timing; it is the visual argument. When utilization is moderate, variability is absorbed. Near saturation, variability has nowhere to go except into waiting.

What page 02 should become

The full version should become a step-through animation: release work, watch it meet a constraint, then switch from local utilization metrics to flow metrics. The contrast should be visible before it is explained.

A Theory of Constraints layer can then mark the constraint, add a buffer, and show why subordinating release to the constraint improves flow even when some non-constraint resources look less busy.
